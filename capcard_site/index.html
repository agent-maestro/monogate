Machine-Readable AI Capability Cards

CapCard is an open JSON standard for publishing what your software can do — in a format that AI agents, orchestrators, and discovery systems can parse, verify, and act on.

See an example Register your card

What is CapCard?

When an AI agent needs to solve a problem, it searches for tools. Today that means reading documentation, running demos, and guessing. CapCard gives every software package a structured JSON file that describes capabilities, benchmarks, and reproduction steps — so agents can discover and use software reliably, without reading prose.

🤖

Agent-Discoverable

Standardized JSON at a predictable URL. Any AI agent can fetch, parse, and reason about your capabilities.

✅

Verifiable

Cards include reproduction commands. Agents can re-run benchmarks to confirm claims before trusting them.

🔍

Searchable

A registry of CapCards lets agents find the right tool for any task — by capability, not by keyword.

📐

Schema-Validated

JSON Schema at /schema/v1.json ensures cards are well-formed and extensible.

Example — monogate CapCard

monogate is the reference implementation: a symbolic computation library built on the EML operator. Its CapCard describes capabilities, benchmarks, and how to reproduce results. 

"$schema": "https://capcard.ai/schema/v1.json",
"card_version": "1.0",
"name": "monogate",
"version": "2.1.1",
"tagline": "Universal EML tree arithmetic for symbolic computation",
"capabilities": {
  "test_count": 1530,
  "sin_barrier": {
    "max_n": 7,
    "sin_absent": true
  },
  "eml_fourier": {
    "sin_floor_n4": 7.79e-8,
    "verdict": "DENSE"
  },
  "identity_theorem": {
    "depth": 4,
    "formula": "eml(1,eml(eml(1,eml(x,1)),1)) = x"
  }
},
"reproduce": {
  "install": "pip install monogate",
  "test": "cd python && pytest tests/ -q"
},
"artifacts": {
  "arxiv": "2603.21852",
  "pypi": "monogate==2.1.1",
  "github": "https://github.com/artl/monogate"
}
📄 Schema — schema/v1.json

Schema Fields

FieldTypeRequiredDescription
$schemastringnoPoints to this schema URL
card_versionstringyesCapCard spec version (currently "1.0")
namestringyesPackage or project name
versionstringyesSoftware version (semver)
taglinestringyesOne-line description, max 120 chars
capabilitiesobjectyesDomain-specific capability claims
benchmarksobjectnoQuantitative benchmark results
reproduceobjectnoCommands to install and reproduce results
artifactsobjectyesLinks to arxiv, pypi, github

Register Your Card

Add a capability_card.json to your repo root and open a PR to the registry. Cards are validated against schema/v1.json automatically.

1️⃣

Create your card

Add capability_card.json to your repo root. Use the monogate card as a template.

2️⃣

Validate it

Run python -m monogate capability-card --validate or use the online validator (coming soon).

3️⃣

Submit to registry

Open a PR at github.com/artl/capcard-registry (coming soon).
